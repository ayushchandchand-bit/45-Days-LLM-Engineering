Foundations

What is Software Engineering?

It is not just "writing code." Coding is one person writing a script that works once. Engineering is teams building products that work for millions, for years.

👩‍💻

Just coding

One file. Runs on my laptop. Works today. Nobody else reads it.

🏗️

Software engineering

Correct, reliable, secure, maintainable. Built by teams. Tested, deployed, monitored, improved.

The extra words — reliable maintainable scalable — are what make it engineering.

Softpro School of AICoding ≠ Engineering

The two sides

Client & Server

Every app has two halves talking over the internet:

📱 Client your phone / browser
⇄
🌐 Internet requests & responses
⇄
🖥️ Server a computer elsewhere
  • →Client (frontend) runs on your device. Shows the app.
  • →Server (backend) runs in a data center. Does the heavy lifting & remembers things.
Softpro School of AITwo computers, one conversation
Frontend

Frontend — what the user sees 👀

The screen, buttons, forms, colors, animations. If you can click it or read it, it's the frontend.

🧱

HTML

Structure — the skeleton

🎨

CSS

Style — looks & layout (this very slide is HTML + CSS!)

⚡

JavaScript

Behavior — interactivity

Frameworks: React, Vue, Angular. In this course: we'll use Streamlit to build UIs in pure Python — no JS needed.

Softpro School of AI"If you can click it, it's frontend"
Backend

Backend — the brain behind the scenes 🧠

You never see it, but nothing works without it. It runs on the server and handles the logic, rules, security, and data.

  • →Processes requests ("log this user in", "summarize this PDF")
  • →Enforces rules & security (passwords, permissions, payments)
  • →Talks to the database and to other services / AI models

Languages: Python, Node.js, Java, Go. In this course: Python — the language of AI.

Softpro School of AIInvisible, but in charge
API

API — how the pieces talk 🤝

Application Programming Interface = the waiter. A clear contract: send me this, I'll send you that.

# Frontend asks the backend a question over HTTP GET /weather?city=Lucknow # Backend replies with structured data (JSON) { "city": "Lucknow", "temp": 41, "unit": "C" }

⭐ This is the big one for you: calling AI is just calling an API. send a prompt → get text back. You'll make your first Groq API call on Day 9.

Softpro School of AIThe contract between programs
Database

Database — where data lives forever 🗄️

Close the app, the data's still there: your account, posts, orders, messages. The pantry that never forgets.

📊

SQL (tables)

Rows & columns, like a spreadsheet. PostgreSQL, MySQL

📄

NoSQL (documents)

Flexible, JSON-like. MongoDB, Redis

🤖 For AI you'll meet a new kind: Vector DBs (Chroma, pgvector) — they store meaning, not just text. That's Phase 2.

Softpro School of AIThe memory that outlives the app
Putting it together

What happens when you tap ❤️ on a post?

1
Frontend: you tap the heart — the screen sends a request
2
API: a request travels over the internet to the server
3
Backend: checks you're logged in, increases the like count
4
Database: saves "this user liked this post" permanently
5
Response: backend replies "done ✅"
6
Frontend: the heart turns red — all in ~200 milliseconds
Softpro School of AIThe full round-trip, every single time
Version Control

Git & GitHub — save points + teamwork 🔁

Git is like a "save game" for code — but smarter. It records every change so you can rewind, branch, and let a whole team work without overwriting each other.

📸

Commit

A snapshot of your code at a moment

↩️

History

Undo to any past version

👥

Collaborate

Many people, one project, no chaos

Git = the tool · GitHub = its home in the cloud. You'll finish this course with 4 projects on GitHub — your portfolio.

Softpro School of AINever lose work. Never overwrite a teammate.
Cloud

The Cloud ☁️

The cloud is just someone else's computer — that you rent instead of buy.

  • →No buying servers — rent power by the minute
  • →Scale from 1 user to 1 million without new hardware
  • →Big players: AWS, Google Cloud, Azure

💸 In this course everything is free-tier: Hugging Face Spaces Render Vercel Supabase — you'll deploy real apps at ₹0.

Softpro School of AIRent, don't buy
DevOps

DevOps — building the bridge 🌉

Not a tool — a culture & set of practices that joins two worlds that used to fight:

🛠️ Dev build the software
+
🤝 DevOps automate the handoff
+
⚙️ Ops run it in production

Goal: ship faster and more reliably by automating the boring, error-prone steps between "it works on my laptop" and "it's live for users."

Softpro School of AIDev + Ops = ship with confidence
CI/CD

CI/CD — the automation pipeline 🚀

CI = Continuous Integration · CD = Continuous Delivery. You push code; a robot does the rest.

⌨️ Code you write
→
⬆️ Push to GitHub
→
🔨 Build assemble app
→
✅ Test auto-check
→
🌍 Deploy live to users

Every change is tested and shipped automatically — no manual, midnight, error-prone deploys.

Softpro School of AIPush code → robot ships it
A day in the life

A day in the life of a Software Engineer 🗓️

You don't just "code all day." Work flows as tickets on a board (Jira) that travel from an idea to a live feature. Here's one ticket's journey:

📋 To Do

PROJ-42
☕ Standup: pick a ticket — "Add a Login button"

🔨 In Progress

🌿 Branch: git checkout -b feature/login
⌨️ Build it + write tests

👀 In Review

🔀 Pull Request: teammate reviews your code
✅ CI runs all tests automatically

✅ Done

🚀 Merge → CD deploys to production
📈 Monitor · drag ticket to Done

Every feature you've ever used started as a ticket that travelled across a board like this.
