# Phase 1 — Foundations (Days 1–15)

**Goal:** Get everyone to baseline. By the end of this phase you'll have your first working,
deployed LLM app.

> 🐍 **New to Python? Set it up first:** [**Python Installation Guide**](Python-Installation-Guide.md)
> — Windows, macOS & Linux, with verification, virtual environments, and troubleshooting.

## Days 1–7 — Python power-up
A from-scratch Python foundation. Even if you know some Python, these days lock in the exact
skills the AI track depends on (clean functions, data structures, error handling, OOP).

- **[Day 01](Day-01-Python-Setup-Variables-and-Data-Types/)** — Setup, data types, operators & variables
- **[Day 02](Day-02-Strings-F-Strings-and-Methods/)** — Strings, f-strings & string methods
- **[Day 03](Day-03-Booleans-and-Conditionals/)** — Booleans, conditionals & logical operators
- **[Day 04](Day-04-Loops/)** — Loops
- **[Day 05](Day-05-Functions-and-Scope/)** — Functions & scope
- **[Day 06](Day-06-Data-Structures/)** — Data structures (lists, dicts, tuples, sets)
- **[Day 07](Day-07-Errors-Modules-and-OOP/)** — Errors, modules & OOP

## Days 8–15 — First LLM app
- **Day 08** — Python for AI: `requests`, async basics, `.env` config, Pydantic, type hints
- **Day 09** — LLM fundamentals (tokens, context, sampling) + **first Groq call**
- **Day 10** — Prompt engineering I: system prompts, zero-/few-shot, chain-of-thought
- **Day 11** — Prompt engineering II: structured outputs with Pydantic, JSON mode
- **Day 12** — Multi-provider patterns: Groq / Ollama / Gemini behind one abstraction
- **Day 13** — Working with data: JSON, CSV, PDF, lightweight scraping
- **Day 14** — Streamlit crash course + error handling, retries, rate limits, cost
- **Day 15** — Mini-project 1 build + deploy + demo

**Mini-project 1 — AI Content Summarizer:** PDF / article URL / transcript in → structured
summary (key points, TL;DR, action items) out. Stack: Streamlit + Groq + Python. Deployed free.
